Step 2: The 5 most common real causes (in order)
1️⃣ Email code is NOT executing at all

This is very common after role / flow changes.

Check immediately
Add a hard log before sending email:

console.log("📧 EMAIL_TRIGGERED", {
  to,
  type: "invite | prospect",
});


If you don’t see this log:

The email function is never being called

Bug is in flow logic, not Resend

2️⃣ Resend API key is missing in production

Replit secrets must exist in Production, not just Dev.

Verify
In Production App Secrets:

RESEND_API_KEY ✅

RESEND_FROM_EMAIL ✅

Then log at runtime:

console.log("RESEND KEY PRESENT", !!process.env.RESEND_API_KEY);


If false → that’s the bug.

3️⃣ From email domain is NOT verified in Resend

Resend will silently drop emails if domain isn’t verified.

Example problem:

RESEND_FROM_EMAIL = noreply@yourdomain.com


But domain not verified in Resend dashboard.

Fix

Go to Resend dashboard

Verify the domain

Or temporarily use:

onboarding@resend.dev


(for testing)

4️⃣ Email promise is failing but error is swallowed

This is extremely common.

Bad code:

resend.emails.send({...});


Correct code:

try {
  const response = await resend.emails.send({...});
  console.log("EMAIL SENT", response);
} catch (err) {
  console.error("EMAIL FAILED", err);
}


If you don’t await, failures disappear.

5️⃣ App crashes before email send completes

If you:

Send email

Immediately return response

Or exit process

Email may never send.

Make sure:

Email send is awaited

No early return
