FRONTEND MAPPING FOR AI RECOMMENDATION & SELECTION LOGIC

I’ll break this into screens, components, states, and data flow.

1️⃣ COMPOSE MESSAGE SCREEN (MOST IMPORTANT)
📍 Screen

Path:
/compose
/campaign/:id/compose
/thread/:id/reply

🧱 Components to Add
A. AIRecommendationPanel (NEW — TOP PRIORITY)

Position:
👉 Above the message editor
👉 Always visible before typing

Component Structure
<AIRecommendationPanel>
  <Badge>AI Recommendation</Badge>
  <Title>{template_name}</Title>
  <Reason>{why_this_works}</Reason>
  <Warning>{avoid_this_mistake}</Warning>

  <PrimaryCTA>Use This Message</PrimaryCTA>
  <SecondaryCTA>See Alternative</SecondaryCTA>
</AIRecommendationPanel>

When This Component Loads (Trigger)

On page load

On thread open

On clicking “Follow-up”

On changing campaign stage

Data Passed to AI
{
  user_role,
  campaign_stage,
  channel,
  days_since_last_touch,
  last_message_text,
  prospect_reply,
  reply_type,
  trigger_detected,
  ICP_type,
  template_history
}

User Actions

Use This Message

Inserts AI-selected message into editor

See Alternative

Replaces content with backup option

Manual Edit

Allowed, but validated before send

Guardrail Hook (Before Send)

Attach to:

onSendClick()


If AI flags risk → block send + show warning modal.

2️⃣ MESSAGE EDITOR (ENHANCED WITH AI VALIDATION)
🧱 Component

MessageEditor

New AI Hooks
A. PreSendValidation()

Runs checks:

Length > 130 words

Multiple CTAs

Repeated CTA

Pitch in first touch

Calendar link in first email

UX Example
⚠️ AI Warning
This message sounds like a pitch and may reduce reply rates.

[Fix with AI]  [Send Anyway]


(Managers can disable “Send Anyway” for SDRs)

8️⃣ FRONTEND EVENT → AI FLOW (SIMPLE)
User Action
  ↓
Frontend emits context
  ↓
AI Recommendation Engine
  ↓
Structured response
  ↓
UI renders recommendation
  ↓
User accepts / edits
  ↓
AI validates
  ↓
Send

FINAL PRODUCT TRUTH (VERY IMPORTANT)

If this logic:

Isn’t visible

Isn’t interruptive at the right moment

Isn’t decisive

Users will ignore it.

You’re not building “AI features”.
You’re building decision support at the moment of action.
