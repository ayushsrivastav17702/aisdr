🔹 FR-A4: USER ADMINISTRATION TESTING
A. User Lifecycle Management

Perform the following as Admin:

Add a new user (single user)

Verify successful login

Bulk upload users via CSV (5+ users)

Verify partial failure handling

Remove a user

Verify immediate access revocation

❌ Flag if bulk upload is missing or fails silently.

B. Role Assignment

Change a user role (User → Manager)

Log in as that user

Verify permission change reflects immediately

❌ Flag if refresh or re-login is required.

C. Password & Access Control

Reset user password

Force password change on next login

Disable user account

Attempt login → must fail

Re-enable → login must succeed

D. Activity Logging

Verify logs exist for:

Login

Logout

Role change

Password reset

Account disable/enable

Check:

UI visibility

Backend log persistence

❌ Flag if logs exist only on frontend or are incomplete.

E. License Allocation

Assign license to user

Attempt over-allocation

Disable user → verify license released

F. Profile & Metadata

Edit:

Name

Team

Territory

Custom metadata fields

Refresh → confirm persistence.

🔹 FR-A5: ROLE-BASED ACCESS CONTROL (RBAC)
A. Custom Role Creation

Create role: Campaign Analyst
Permissions:

Campaign: View only

Prospect: View only

Analytics: View + Export

Settings: None

User management: None

B. Permission Enforcement Matrix

Log in as Campaign Analyst and test:

Action	Expected
Create campaign	❌
Edit campaign	❌
Delete campaign	❌
View campaign	✅
Export analytics	✅
Delete prospect	❌
Access settings	❌

❌ Flag if UI blocks but API allows.

C. User-Level Overrides

Override permission: allow “Create Campaign”

Log in → verify only that permission is added

Remove override → permission revoked

D. Permission Inheritance

Deny permission at workspace level

Allow at team level

Validate effective permission

Repeat inverse test.

E. Security Validation (CRITICAL)

Using DevTools or API client:

Call restricted APIs as low-privilege user

Expect 403 Forbidden

❌ If API allows → RBAC FAILURE
