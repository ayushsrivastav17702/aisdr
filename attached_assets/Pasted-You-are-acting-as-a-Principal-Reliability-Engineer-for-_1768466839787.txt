1️⃣ DATABASE FAILURE RESILIENCE
Problem

Writes fail or are lost when DB or read replica is unavailable.

REQUIRED IMPLEMENTATION
Write-Ahead Buffer

Before any DB write:

Persist operation to write_ahead_log (WAL) table OR Redis-backed durable queue

Mark as pending

On success → mark committed

On failure → retry via background worker

Read Replica Failure

Auto-detect replica errors

Failover reads to primary

Never block writes due to read failure

UI Behavior

User sees:
“Temporary database issue. Action queued and will auto-retry.”

Tests to Add

Kill read replica → writes still succeed

Kill primary DB → writes queued

Restart DB → queued writes flush correctly
