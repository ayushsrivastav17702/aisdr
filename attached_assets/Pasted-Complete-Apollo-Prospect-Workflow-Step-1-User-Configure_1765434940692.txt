Complete Apollo Prospect Workflow
Step 1: User Configures ICP Filters (Frontend)
File: client/src/components/icp-prospect-filters.tsx

User selects job titles, industries, company sizes, locations
Filters stored in React state
Options include: SaaS, Technology, Healthcare, VP, Director, C-Suite, etc.
Step 2: Frontend Sends Search Request
// User clicks "Search Prospects" button
apiRequest('POST', '/api/apollo/search-prospects', {
  jobTitles: ['VP Sales', 'Sales Director'],
  industries: ['SaaS', 'Technology'],
  companySizes: ['51-200', '201-500'],
  locations: ['United States'],
  limit: 100
});

Step 3: Backend Applies Smart Defaults
File: server/services/apollo.ts

If user didn't specify certain filters, system applies ICP defaults
Default job titles: CEO, CTO, VP Sales, Sales Director, etc.
Default industries: Technology, SaaS, Healthcare, Retail
Default locations: United States, Canada, UK
Step 4: Backend Converts to Apollo API Format
// User-friendly → Apollo API format
{
  person_titles: ['VP Sales', 'Sales Director'],      // ← jobTitles
  organization_industries: ['SaaS', 'Technology'],    // ← industries
  organization_num_employees_ranges: ['51-200'],      // ← companySizes
  person_locations: 'United States',                  // ← locations
  per_page: 100
}

Step 5: Apollo API Returns Verified Data

Step 6: Backend Formats Prospect Data
// Transform to app's data structure

Step 7: Store in Database (Duplicate Check)
// Check if email already exists
const exists = existingProspects.some(p => 
  p.email.toLowerCase() === prospect.email.toLowerCase()
);
if (!exists) {
  await storage.createProspect(newProspect);  // Save to database

Fallback Strategies (5 Auto-Retry Patterns)
If the initial search returns 0 results, the system automatically tries:

Core Terms: buyer, merchandiser, purchasing manager
E-commerce Focus: ecommerce, digital commerce, online retail
Retail Ops: retail manager, inventory manager, product manager
Supply Chain: procurement, logistics, vendor manager
Broader Terms: merchandising, retail, product, inventory
