🧱 PART 1 — RETRY & WATCHDOG
FILE: /server/queue/retry.service.ts

Create function:

export async function retryFailedEmails()


Logic:

Query:

SELECT * FROM email_queue
WHERE status = 'failed'
AND retry_count < 3
AND retryable = true
AND tenant_id IS NOT NULL


For each:
• Compute delay:
retry_count = 0 → 5 min
retry_count = 1 → 15 min
retry_count = 2 → 60 min

Update:

UPDATE email_queue
SET status='pending',
retry_count = retry_count + 1,
next_retry_at = NOW() + INTERVAL 'X minutes'
WHERE id = ?


Insert into retry_logs:

{
  email_queue_id,
  attempt,
  reason,
  timestamp
}


Never mark sent without messageId.

FILE: /server/queue/watchdog.service.ts

Create function:

export async function scanForStuckEmails()


Definition of stuck:
• status = 'pending'
• created_at < NOW() - INTERVAL '10 minutes'
• no worker heartbeat

Query:

SELECT * FROM email_queue
WHERE status='pending'
AND created_at < NOW() - INTERVAL '10 minutes'


For each:
• Update:

UPDATE email_queue SET status='stuck'


• Insert audit_log:

event_type='QUEUE_STUCK'


• Call alert service.

Schedule every 5 minutes with existing scheduler.
