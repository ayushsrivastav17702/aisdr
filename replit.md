# AI-Powered SDR Platform

## Overview
An AI-powered Sales Development Representative (SDR) platform designed to streamline prospect discovery, enrichment, and outreach. It converts natural language queries into structured Apollo.io searches, enriches prospect data, and automates multi-step email sequences with AI-generated, personalized content. The platform aims to enhance sales efficiency by providing a comprehensive solution for lead generation and engagement, from initial search to automated email follow-ups and reply tracking.

## System Architecture
The platform is built with a modern web stack:
- **Frontend**: React, TypeScript, Vite, Tailwind CSS, shadcn/ui for a responsive and intuitive user interface.
- **Backend**: Express.js with TypeScript for robust API services.
- **Database**: PostgreSQL (managed by Neon) with Drizzle ORM for data persistence, storing prospects, searches, jobs, and comprehensive email sequence data including mailboxes, queues, and send logs.
- **AI Integration**: Leverages OpenAI (GPT-4o, GPT-5) or Anthropic (Claude Sonnet 4) for natural language processing, personalized email generation, LinkedIn analysis, and sentiment analysis.
- **Job Queue**: Utilizes BullMQ (requires Redis/Upstash) for background processing of tasks like enrichment, CSV imports, and email sending.
- **UI/UX Decisions**: Employs a clean, modern design with a focus on user-friendly workflows for AI search, prospect management, and campaign creation. Key features include a detailed email sequence builder with real-time reply detection, AI-powered email generation with customizable tones, and multi-mailbox email sending with warmup and round-robin capabilities.
- **Technical Implementations**:
    - **Natural Language Processing**: Converts user queries into structured Apollo.io filters with intelligent AI and fallback processing.
    - **AI Search Fallback**: When AI providers fail, uses keyword-based extraction that retains company names (including short/digit-leading like IBM, Gap, 3M, 23andMe), job titles (including merchandising roles), locations, and industries while filtering common stop-words and avoiding false positives from locations or sentence starters.
    - **Apollo Industry Filter Fix**: Industries are included in keyword search (`q_keywords`) instead of `organization_industry_tag_ids` to prevent 422 errors from invalid industry tag IDs. This ensures AI search works reliably even without exact Apollo.io industry mappings. Invalid industry IDs previously caused "Your search parameters are entered incorrectly" errors, now resolved by using keyword-based industry matching.
    - **Email Sequence Management**: Supports multi-step sequences, prospect enrollment, tracking, and AI-powered personalization based on prospect data.
    - **AI Email Generation**: One-click generation of personalized emails, A/B test variants, and sentiment analysis.
    - **Multi-Mailbox Sending**: Securely manages multiple email accounts (Gmail, Outlook, SMTP, SendGrid) with round-robin rotation, encryption, and an email warmup system.
    - **Bulk Operations**: Efficiently enriches multiple prospects simultaneously using Apollo's bulk match API.
    - **Data Security**: Secure credential encryption (AES-256-CBC with random IV) for email mailboxes.
    - **API Key Management**: Automatic fallback from primary to backup OpenAI API key when quota is exhausted (429 errors).
    - **CSV Import Resilience**: 50MB file size limit, comprehensive logging for debugging large imports (15k+ prospects), detailed error tracking with row numbers.
    - **Reply Detection System**: IMAP-based polling (every 20 seconds) that automatically detects prospect replies, matches them to sent emails, and stores them with proper error handling. Uses promise-based async handling to ensure all replies are processed before marking emails as seen. Batch-marks emails with \Seen flag only after successful database save to prevent data loss on errors. Implements intelligent reply content cleanup that strips quoted text, signatures, and email threads to show only actual prospect responses. Prioritizes In-Reply-To header matching for accurate sequence assignment.
    - **AI Follow-up Generation**: Uses actual reply content from database instead of generic templates. Fetches latest reply from email_replies table and joins with sent email for full context before generating contextual follow-ups that respond to what the prospect actually said.
    - **Content Library Auto-Selection**: Automatically selects all content library items when PersonalizationWizard opens, ensuring content library validation always runs. UI clearly indicates "Auto-selected for Compliance" to users. This guarantees that all AI-generated emails use approved Increff merchandising content and follow brand guidelines.
    - **Content Library Validation with Auto-Retry**: Advanced validation system that **enforces** content library compliance when generating AI emails. System automatically retries generation up to 3 times if violations are detected. Detects forbidden generic phrases (e.g., "multi-brand operations", "integrated management"), validates that Increff-specific solutions are mentioned (e.g., "Assortment Planning & Buying", "Markdown Optimization"), and ensures only approved statistics from content library are used (13%, 36%, 26%, 7%). If all retry attempts fail validation, system returns an error instead of non-compliant email. This **forces AI compliance** rather than just warning users.
    - **Auto-Enrollment from AI Personalization**: When creating an AI personalized email for a prospect in a sequence step, the prospect is automatically enrolled in the sequence. This streamlines the workflow by eliminating the need to manually enroll prospects after creating personalized content. The system maintains UUID string integrity throughout the enrollment process, preventing type conversion issues.
    - **Performance Optimizations**: Prospects table page size reduced from 50 to 25 items for improved rendering performance and reduced lag. Implements animated skeleton loading states (8 pulsing placeholder rows) during data fetches to provide visual feedback and prevent UI freezing. URL parameter synchronization keeps page state in sync with browser URL (e.g., /?page=2), enabling direct navigation and proper back/forward button support. Invalid page parameters (NaN, negative, zero) automatically default to page 1 to prevent API errors.
    - **Sequence Management**: Complete sequence deletion feature with CASCADE delete constraints ensures clean removal of sequences and all related data (email_queue, sequence_steps, sequence_prospects, email_replies). Delete buttons appear on hover in both grid and list views with confirmation dialogs to prevent accidental deletion. Successfully handles edge cases like sequences with active email queues and pending sends.
    - **High-Speed Email Sending**: Email queue processor runs every 10 seconds (reduced from 60 seconds) for near-instant email delivery. Replies are sent within 10 seconds of user action. Reply composer dialog allows users to generate AI follow-ups, edit content, and send directly through the platform instead of external email clients. Success notifications confirm queuing with delivery timeline. Auto-refresh intervals reduced to 20 seconds for faster UI updates.
    - **Email Formatting for HTML Clients**: AI-generated emails with proper paragraph spacing rendered correctly in Gmail and other HTML email clients. System wraps each paragraph in HTML `<p>` tags with 16px bottom margin for optimal visual separation. Email sending service splits content by line breaks and wraps each paragraph in `<p style="margin: 0 0 16px 0;">` tags, ensuring emails display with clear visual separation between sections (opening, problem statement, solution, call-to-action). This replaced the previous `<br>` tag approach which didn't create sufficient spacing in email clients.
    - **AI Reply Composer**: Fully functional reply dialog allows users to generate AI follow-ups and send replies directly through the platform. Send Reply button properly queues emails with high priority for near-instant delivery. Mutation includes prospectId, sequenceId, subject, and body parameters as required by backend API.
    - **Email Analytics Tracking**: Email analytics (sent, delivered, opened, replied counts) now properly track all sent emails. Queue processor creates entries in emails table when sending, setting sentAt timestamp and trackingId for correlation. Sequence analytics display accurate statistics for all email activity.
    - **Email Signatures**: Mailboxes support customizable email signatures that are automatically appended to all outgoing emails. Signatures are stored in the email_mailboxes table and formatted as HTML paragraphs with reduced spacing (4px bottom margin) and gray color (#666) for visual distinction from email body. Signature for Kanika Bhalla (CRO) includes name, title, location, LinkedIn, and phone contact information.
    - **Contextual AI Follow-ups**: AI-generated follow-up emails now properly reference and build upon previous email conversations instead of starting fresh. System formats previous email history with clear labels (Email 1, Email 2, etc.) separated by dividers, ensuring AI has full conversation context. Enhanced FOLLOW_UP_PROMPT explicitly requires AI to reference previous email topics, build naturally upon discussed content, and include "Re:" subject prefix. Prevents generic "just checking in" messages by enforcing contextual continuity.
    - **Sequence Builder Performance**: Lazy loading optimization eliminates UI lag when opening sequences. Only sequence details load initially; prospects and replies data load on-demand when their respective tabs are clicked. This prevents blocking the main thread with large Apollo enrichment data. Sequence builder now opens instantly (measured at 13ms) compared to previous hanging behavior. Replies polling (30s interval) only runs when Replies tab is active, reducing unnecessary API calls.

## External Dependencies
- **Apollo.io**: Primary API for prospect search, data enrichment, and bulk matching.
- **OpenAI**: Provides AI capabilities for natural language processing, email generation (GPT-4o, GPT-5), and LinkedIn personalization.
- **Anthropic**: Alternative AI provider (Claude Sonnet 4) for natural language processing.
- **Lusha.io**: Used for email enrichment, serving as a fallback when Apollo.io data is unavailable or locked.
- **PostgreSQL (Neon)**: Cloud-hosted relational database for all application data.
- **Redis/Upstash**: Required for BullMQ to enable background job processing and asynchronous task management.
